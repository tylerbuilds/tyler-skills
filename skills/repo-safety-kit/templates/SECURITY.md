# Security Policy

## Reporting Security Issues

Please report security issues privately. Do not open public issues containing secrets, exploit details, private logs, or live incident data.

## Secret Handling

This repository must not contain:

- API tokens or API keys.
- Passwords or private keys.
- Raw `.env` values.
- Service-token secrets.
- Production logs, exports, screenshots, or user data.

Use placeholders such as `<api-token>`, `<account-id>`, and `example.com`.
